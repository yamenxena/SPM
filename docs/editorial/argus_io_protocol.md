## 3. Data Models

### 3.1 Atom (Single Extracted Fact)

```json
{
  "id": "ATOM_001",
  "who": "Abu Seif",
  "who_canonical": "Abu_Seif",
  "who_type": "PERSON",
  "did_what": "tortured",
  "to_whom": "Talal al-Shweimi",
  "to_whom_canonical": "Talal_Mustafa_al_Shweimi",
  "where": "Rumaila Prison",
  "where_canonical": "RUMAILA_PRISON",
  "when": "summer 2014",
  "when_iso": "2014-06/2014-09",
  "sensory": ["Temperature:heat"],
  "source_file": "01_prison room_En.srt",
  "source_cue": 42,
  "source_hash": "a1b2c3d4...",
  "confidence": "stated"
}
```

### 3.2 Edge (Graph Connection)

```json
{
  "id": "EDGE_001",
  "type": "PERPETRATED",
  "from_node": "Abu_Seif",
  "to_node": "EVT_RUMAILA_001",
  "source_atoms": ["ATOM_001", "ATOM_007"],
  "requires_human_gate": true,
  "human_approved": false,
  "corroboration_score": 1,
  "corroboration_tier": "UNCORROBORATED"
}
```

### 3.3 Node (Graph Entity)

```json
{
  "id": "Abu_Seif",
  "type": "PERSON",
  "sub_type": "PERPETRATOR",
  "canonical_name": "Abu Seif",
  "arabic_name": "أبو سيف",
  "glossary_match": true,
  "glossary_source": "5_glossary_isis.md",
  "aliases": ["Abu Sayf", "ابو سيف"],
  "attributes": {
    "role": "Emir",
    "facility": "RUMAILA_PRISON",
    "domain": "isis"
  },
  "first_seen_in": "01_prison room_En.srt",
  "corroboration_score": 1,
  "corroboration_tier": "UNCORROBORATED"
}
```

**Valid `sub_type` values for PERSON:** WITNESS, PERPETRATOR, CO_DETAINEE, VICTIM, BYSTANDER

**Derivation rule:** `sub_type` is derived from edges:
- Has WITNESSED edges (as source) → `WITNESS`
- Has PERPETRATED edges (as source) → `PERPETRATOR`
- Has CO_DETAINED edges → `CO_DETAINEE`
- Is target of PERPETRATED → `VICTIM`
- Default → `PERSON`

## 5. Graph Integrity: Witness Nexus Rule (G-10)

Every node in the FKG **MUST** be reachable from the primary witness through an unbroken chain of edges. This is enforced by `validate_connectivity()` in `validation_gate.py`.

### HALT Condition
If `validate_connectivity()` detects disconnected components (islands), the pipeline **HALTs** with exit code 1. The operator must add linking edges before rerunning.

### Linking Edge Types
Use these edge types to connect orphaned clusters back to the main graph:

| Edge Type | When To Use | Example |
|-----------|-------------|---------|
| `WITNESSED` | Witness testified about this event | Talal → EVT_REGIME_DISMISSAL |
| `DETAINED_AT` | Person was held at this facility | Talal → POINT_11 |
| `MEMBER_OF` | Person worked at/belonged to org | Abu_Yousuf → MIL_JUDICIARY |
| `LOCATED_IN` | Facility is part of another facility | Solitary_Cells → RUMAILA_PRISON |
| `OCCURRED_AT` | Event happened at this location | EVT_SEIZURE → HISBA_SAJIYAH |

### Why This Rule Exists
In forensic testimony graphs, a disconnected cluster means the LLM extracted facts but failed to connect them to the witness's chain of knowledge. If a witness testifies about an event, the graph must show **how** the witness knows about it (direct experience, organizational proximity, or spatial co-location). Without this link, the evidence chain is broken and the graph is forensically dishonest.

## 6. G-11: Cross-Witness Evidence Rule

CROSS_WITNESS edges **MUST** connect shared **concept nodes**, **NEVER** witness-to-witness.

### Rationale
A direct edge between two witnesses is forensically meaningless — two people testifying about the same prison does not mean they know each other. The evidential value lies in the **shared concept** (same torture method, same facility, same execution pattern) confirmed independently by multiple sources.

### Valid CROSS_WITNESS Edges

| From (W1 concept) | To (W2 concept) | `common_edge` | Why Valid |
|-------------------|-----------------|---------------|----------|
| `SHABEH_SUSPENSION` | `SHABEH_BALINGO_SUSPENSION` | SHABEH | Same torture method, different prisons |
| `BEATING_STICKS_METAL_RODS` | `BEATING_HOSE_STICK` | BEATING | Same beating practice, independent reports |
| `EVT_QISAS_SENTENCING` | `EVT_W2_QISAS_8_MANSOURA` | QISAS | Same judicial execution pattern |

### Invalid CROSS_WITNESS Edges

| From | To | Why Invalid |
|------|----|-------------|
| `Talal_al_Shuweimi` | `Faraj_Salamah` | Witnesses are not relatives or cellmates |

### Exception
Direct edges between witnesses are permitted **only** with types `CO_DETAINED` or `RELATED_TO`, when testimony confirms shared detention or kinship.
